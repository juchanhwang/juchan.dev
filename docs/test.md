군선교가 아니라 모든 선교 관리 프로젝트야.
프로젝트를 시작하게 된 계기는 기존 선교의 시스템과 프로세스의 문제를 인식했기 때문이야. 
선교 기간이 되면 각 선교를 준비하는 준비팀들이 선교를 각각 준비해.
1. 선교에 필요한 모든 프로세스는 파편화된 툴을 사용해. 
2. 관리자 입장에서 하나의 시스템에서 볼 수 없는 문제가 있어.
3. 자신이 맡은 일이 아니면 확인이 어려운 문제가 있어.
4. 모든 선교팀의 공통 포맷도 아니여서 교회차원에서 통일성의 문제도 있어.
모든 파편화된 시스템을 하나의 플랫폼에서 통일하자는 취지에서 시작되었어.
