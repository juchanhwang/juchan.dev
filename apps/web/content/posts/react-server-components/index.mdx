## Server Components란?

React Server Components(RSC)는 서버에서만 실행되는 컴포넌트입니다. 클라이언트로 JavaScript 번들이 전송되지 않아 성능상 이점이 큽니다.

Next.js App Router에서는 **기본적으로 모든 컴포넌트가 Server Component**입니다.

```tsx title="app/blog/page.tsx"
// 이 컴포넌트는 서버에서만 실행됩니다
import { posts } from "#velite";

export default function BlogPage() {
  const publishedPosts = posts.filter((post) => !post.draft);

  return (
    <main>
      <h1>블로그</h1>
      <ul>
        {publishedPosts.map((post) => (
          <li key={post.slug}>
            <a href={post.permalink}>{post.title}</a>
          </li>
        ))}
      </ul>
    </main>
  );
}
```

## 서버 vs 클라이언트, 어떻게 나눌까?

핵심 판단 기준은 간단합니다:

| 필요한 것 | 컴포넌트 타입 |
|---|---|
| 데이터 fetch, DB 접근 | Server |
| `useState`, `useEffect` | Client |
| 브라우저 API (localStorage 등) | Client |
| 이벤트 핸들러 (onClick 등) | Client |
| 정적 UI 렌더링 | Server |

### 실전 패턴: 경계 설정

```tsx title="components/blog/post-detail.tsx"
// Server Component — 데이터를 가져오고 레이아웃을 구성
import { TableOfContents } from "./table-of-contents";
import { FloatingActions } from "./floating-actions";
import { MDXContent } from "@/lib/mdx";

interface PostDetailProps {
  title: string;
  date: string;
  tags: string[];
  toc: { title: string; url: string; depth: number }[];
  body: string;
}

export function PostDetail({ title, date, tags, toc, body }: PostDetailProps) {
  return (
    <article>
      <header>
        <h1>{title}</h1>
        <time>{date}</time>
        <div>{tags.map((tag) => <span key={tag}>{tag}</span>)}</div>
      </header>
      <div className="relative flex gap-8">
        <FloatingActions />
        <div className="prose max-w-3xl">
          <MDXContent code={body} />
        </div>
        <TableOfContents items={toc} />
      </div>
    </article>
  );
}
```

핵심은 **가능한 한 서버에서 처리하고, 인터랙션이 필요한 부분만 클라이언트 컴포넌트로 분리**하는 것입니다.

## 흔한 실수들

### 1. 불필요한 "use client"

```tsx
// 이벤트 핸들러가 없는데 "use client"를 붙이는 실수
"use client"; // ❌ 불필요

export function StaticCard({ title }: { title: string }) {
  return <div className="card">{title}</div>;
}
```

### 2. 서버 컴포넌트에서 클라이언트 훅 사용

```tsx
// ❌ Server Component에서 useState 사용 불가
import { useState } from "react";

export default function Page() {
  const [count, setCount] = useState(0); // 에러!
  return <div>{count}</div>;
}
```

## 정리

RSC를 잘 활용하면 번들 사이즈를 줄이고, 초기 로딩 성능을 크게 개선할 수 있습니다. "기본은 서버, 필요할 때만 클라이언트"를 기억하세요.
