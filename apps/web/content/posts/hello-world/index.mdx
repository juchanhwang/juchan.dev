## 왜 새로 만들었나

기존 블로그는 Gatsby 2.x 기반이었습니다. EOL된 프레임워크 위에서 빌드가 불안정해지고, 플러그인 생태계 지원이 끊기면서 유지보수가 점점 어려워졌습니다.

포트폴리오 사이트도 별도 프로젝트로 분리되어 있어서, 브랜딩 일관성도 떨어지고 관리 비용도 두 배였습니다.

## 기술 스택 선택

새 블로그의 기술 스택은 다음과 같습니다:

```typescript title="tech-stack.ts"
const stack = {
  framework: "Next.js 16",
  runtime: "React 19",
  content: "Velite + MDX",
  styling: "Tailwind CSS v4",
  highlight: "Shiki (rehype-pretty-code)",
  deploy: "Vercel",
} as const;
```

### Velite를 선택한 이유

Contentlayer가 유지보수 중단된 이후, type-safe한 콘텐츠 관리 도구로 **Velite**를 선택했습니다.

- Zod 기반 스키마로 frontmatter 타입 안전성 확보
- MDX 지원으로 커스텀 컴포넌트 삽입 가능
- 빌드 시 JSON으로 변환되어 런타임 오버헤드 없음

### 코드 하이라이팅

Shiki를 사용해서 VS Code 수준의 문법 하이라이팅을 제공합니다. 다크/라이트 테마를 CSS 변수로 자동 전환합니다.

```tsx {3-5}
function Greeting({ name }: { name: string }) {
  return (
    <div className="rounded-lg bg-background-subtle p-4">
      <h2 className="text-xl font-bold">안녕하세요, {name}님!</h2>
      <p className="text-muted-foreground">블로그에 오신 것을 환영합니다.</p>
    </div>
  );
}
```

- 기존 `julog` 포스트 마이그레이션

- Giscus 댓글 시스템 연동

새 블로그에서 더 좋은 글로 찾아뵙겠습니다.
