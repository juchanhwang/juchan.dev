## 📋 인터뷰 가이드

> 💡 오늘은 어드민 시스템을 만든 팀을 테스트하는 자리입니다. 불편했던 순간이 칭찬보다 훨씬 더 도움이 됩니다. 녹음해도 괜찮으실까요?

### 공통 질문 — 현 선교 시스템

1. 선교 준비팀 또는 선교 대원으로 참여하면서 어떤 순간이 가장 힘드셨어요?
2. 현 선교 시스템의 가장 큰 문제점은 뭐라고 생각하세요? 지금은 그 문제를 어떻게 해결하고 계세요?
3. 매번 선교마다 처음부터 다시 만들어야 하는 것들이 있나요?
4. 막판에 급하게 처리해야 했던 상황이 있었나요? 그때 어떻게 해결하셨어요?

### 섹션 1 — 테스트 경험 재구성 (12분)

1. 테스트를 시작했을 때 제일 처음 하신 행동이 뭐였어요?
2. 테스트하면서 "이게 뭐지?" 하고 멈추셨던 순간이 있었나요? 그때 어떤 화면이었고 뭘 하려고 하셨나요?
3. 원하는 걸 못 찾아서 이리저리 클릭해 보셨던 적 있으세요? 어느 부분이었나요?
4. 테스트 중에 포기하거나 건너뛴 기능이 있었나요? 왜 그러셨어요?

### 섹션 2 — 비교 탐색 (5분)

1. 이 시스템을 쓰면서 "이건 전보다 확실히 빠르다"고 느끼신 순간이 있었나요?
2. 현 시스템을 유지해야 한다고 생각하는 것이 있나요?
3. 마지막으로 하고 싶으신 말씀이 있으실까요?

> 💡 **하면 안 되는 것**: 막힌 이유를 설명하거나 정당화하지 않는다 / "이 기능 어떠세요?" 형태의 평점 요청 금지
>
> **해야 하는 것**: 솔루션 요청 시 → "그게 없어서 지금은 어떻게 하고 계세요?"로 문제로 돌아간다

## 📝 인터뷰 기록

### 핵심 마찰 지점

1. 등록 쿼터 초과 시 팀 내 자체 선착순 링크 별도 운영 (공식 시스템 우회)
2. 준장·등록담당자 외 명단 접근 불가 → 방배정 등 업무 1인 집중
3. 취소·변경 시 버스·식수·여행자보험 연쇄 수작업 처리
4. 영수증 처리 기준이 선교·연도마다 달라 매번 혼선
5. 후기 보고서 이력 없음 — 준비팀 교체 시 데이터 소실
6. 서약서 제출: PDF 수령 → 손서명 → 사진 촬영 → 사이트 업로드 3단계
7. 팀장 공지 읽음 확인 불가 — 이미 공지한 내용 반복 문의 빈번

### 기존 방식 vs 새 시스템

**더 나아진 부분**

- 필터링·정렬 기능 기대 (구글 스프레드시트 대비 직관적, 개인정보로 다운로드 금지인 상황 해소 가능)
- 권한 세분화 시 준비팀원도 명단 접근 가능 → 업무 분산
- UI 디자인 긍정 — "깔끔하고 모서리 둥근 디자인 좋다"

**여전히 기존 방식 선호**

- 카톡 공지 — "카톡 공지만큼 강력한 게 없다" (대체 아닌 보완으로 방향 제안)
- 보고서·서약서 등 파일 제출 — 카톡·이메일이 더 편함
- 구글 폼 — 등록 경험 익숙하고 준비팀도 만들기 편함

### 가장 인상적인 발언

> "어쩔 수 없이 자체적으로 팀 내에서나 진 내에서 자체 등록 링크를 또 만들어"

> "카톡 공지만큼 강력한 게 없음. 근데 대체를 해야 되는 게 아니에요. 방향할 수는 있지"

> "선교 대원들의 개인적 부담감이 늘어나는 건 사실이거든"

### 놀라웠던 발견

- 공식 등록 시스템 신뢰 부족 → 팀 내 자체 링크 병행 운영 (시스템 도입 전제 조건: 등록 확인 신뢰성)
- ESG 명찰 폐지 → 식수 확인 수단 공백 → QR 코드 아이디어가 사용자에게서 자연 발생
- 서약서 3단계 제출 플로우 존재 (사용자 직접 토로)

### OS T 업데이트 후보

- 등록 쿼터 관리 — 팀별 정원 설정 + 선착순 공개
- 등록 상태 실시간 확인 — 등록 완료·입금 확인 본인 조회
- 필터링·정렬 — 전참자/버스별/성별 등 다중 조건
- 권한 세분화 — 준장/준비팀/준비팀장 레벨별 접근 제어
- 푸시 알림 공지 — 카톡 보완용 주요 공지 발송
- 공지 읽음 확인 — 팀장 대상 수신 확인 기능
- 영수증 인앱 업로드 + 선교별 증빙 기준 명시
- 후기 보고서 히스토리 — 연도별 축적 및 열람
- 서약서 전자서명 — 앱 내 서명 처리
- QR 코드 기반 명찰 대체 — 등록 인증 + 식수 확인
